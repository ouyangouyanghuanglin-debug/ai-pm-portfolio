## 1. 现状分析

### 1.1 当前搜索流程
```
用户输入 Query → 关键词匹配 → 返回笔记列表 → 用户逐个查看
```

### 1.2 存在的问题
| 问题 | 描述 | 影响 |
|------|------|------|
| 意图理解差 | 无法理解用户真实需求 | 搜索结果不精准 |
| 结果同质化 | 千人一面，无个性化 | 用户体验差 |
| 信息过载 | 返回大量笔记，用户需筛选 | 决策成本高 |
| 商业干扰 | 广告/种草内容混入 | 信任度下降 |

### 1.3 用户反馈（示例）
- "搜'敏感肌水乳'出来全是广告"
- "想找具体答案，但只能看到一堆笔记"
- "同样的搜索，为什么我和闺蜜看到的不一样？"

## 2. AI 重构方案

### 2.1 核心目标
**从"搜索内容"升级为"获取答案"**

### 2.2 重构架构
```
用户 Query
    ↓
┌─────────────────────────────────────┐
│  意图理解层                          │
│  · 语义分析                          │
│  · 实体识别                          │
│  · 意图分类（查询/推荐/对比/评测）   │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│  检索层                              │
│  · 向量检索（语义相似度）            │
│  · 关键词检索（精确匹配）            │
│  · 混合排序                          │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│  生成层                              │
│  · 大模型摘要生成                    │
│  · 结构化答案输出                    │
│  · 引用来源标注                      │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│  个性化层                            │
│  · 用户画像匹配                      │
│  · 历史行为分析                      │
│  · 实时反馈调整                      │
└─────────────────────────────────────┘
    ↓
返回搜索结果
```

## 3. 功能需求

### 3.1 功能优先级

| 功能 | 优先级 | 工期 | 说明 |
|------|--------|------|------|
| 语义搜索 | P0 | 4 周 | 理解用户真实意图 |
| AI 答案生成 | P0 | 4 周 | 生成结构化答案 |
| 个性化排序 | P1 | 3 周 | 千人千面 |
| 多轮对话 | P1 | 3 周 | 追问澄清 |
| 来源标注 | P1 | 2 周 | 标注答案来源笔记 |
| 商业内容标识 | P2 | 2 周 | 明确标识广告 |

### 3.2 核心功能详情

#### 功能 1：语义搜索

| 项目 | 描述 |
|------|------|
| 功能描述 | 理解用户搜索的真实意图，而非简单关键词匹配 |
| 用户故事 | 作为用户，我希望搜索时能被理解真实需求，以便获得更精准的结果 |
| 输入 | 用户搜索 Query（文本/语音） |
| 处理 | Query 解析 → 意图识别 → 实体抽取 → 向量化 |
| 输出 | 意图标签 + 向量表示 |
| 验收标准 | 1. 意图识别准确率>85% 2. 响应时间<100ms |

#### 功能 2：AI 答案生成

| 项目 | 描述 |
|------|------|
| 功能描述 | 综合多篇笔记内容，生成结构化答案 |
| 用户故事 | 作为用户，我希望直接看到答案摘要，以便快速决策 |
| 输入 | 检索到的相关笔记内容 |
| 处理 | 内容提取 → 信息整合 → 摘要生成 → 格式化 |
| 输出 | 结构化答案 + 来源链接 |
| 验收标准 | 1. 答案准确性>90% 2. 生成时间<3s 3. 来源可追溯 |

#### 功能 3：多轮对话搜索

| 项目 | 描述 |
|------|------|
| 功能描述 | 支持用户追问，AI 主动澄清模糊需求 |
| 用户故事 | 作为用户，我希望通过对话细化需求，以便获得更精准的推荐 |
| 示例 | 用户："敏感肌用什么水乳？" → AI："请问是干性还是油性？预算多少？" |
| 验收标准 | 1. 支持 5 轮以上对话 2. 上下文理解准确 |

## 5. 技术方案

### 5.1 大模型选型
| 方案 | 优点 | 缺点 | 推荐 |
|------|------|------|------|
| 自研模型 | 可控、成本低 | 研发周期长 | 长期 |
| API 调用 | 快速上线 | 成本高、依赖外部 | 短期 |

### 5.2 向量检索
- 使用 Milvus/Faiss 向量数据库
- embedding 模型：bge-large-zh
- 检索延迟：<50ms

### 5.3 成本控制
| 优化手段 | 预期效果 |
|---------|---------|
| Query 缓存 | 减少 30% 调用 |
| 小模型优先 | 简单 Query 用小模型 |
| 结果缓存 | 热门 Query 直接返回 |
