## 1. 产品概述

### 1.1 产品定位
面向职场人的 AI 周报生成工具，整合工作数据，自动生成高质量周报。

### 1.2 目标用户
| 用户类型 | 特征 | 需求 |
|---------|------|------|
| 互联网从业者 | 周报复杂、数据多 | 自动汇总、亮点提炼 |
| 技术人员 | 代码/任务分散 | Git/Jira整合 |
| 产品运营 | 需要数据呈现 | 数据可视化 |
| 管理者 | 需要团队汇总 | 团队周报聚合 |

### 1.3 核心价值
- **节省时间**：从 2 小时→5 分钟
- **提升质量**：结构化、专业化
- **数据驱动**：自动汇总工作数据

## 2. 功能需求

### 2.1 功能优先级

| 功能 | 优先级 | 工期 |
|------|--------|------|
| 手动输入生成周报 | P0 | 3 天 |
| 模板中心 | P0 | 2 天 |
| Git 数据整合 | P1 | 5 天 |
| 飞书/Jira 整合 | P1 | 5 天 |
| 邮件发送 | P1 | 2 天 |
| 历史记录 | P2 | 2 天 |
| 团队周报 | P2 | 5 天 |

### 2.2 核心功能详情

#### 功能 1：周报生成

| 项目 | 描述 |
|------|------|
| 功能描述 | 用户输入工作内容，AI 生成结构化周报 |
| 用户故事 | 作为职场人，我希望快速生成周报，以便节省时间 |
| 输入 | 工作要点（文本/语音） |
| 处理 | 内容理解 → 结构化 → 亮点提炼 → 格式化 |
| 输出 | 完整周报（可编辑） |
| 验收标准 | 1. 生成时间<10 秒 2. 可编辑 3. 支持多轮优化 |

#### 功能 2：数据源整合

| 项目 | 描述 |
|------|------|
| 功能描述 | 自动从 Git/Jira/飞书等平台获取工作数据 |
| 用户故事 | 作为技术人员，我希望自动获取 Git 提交记录，以便周报更准确 |
| 接入平台 | GitHub、GitLab、Jira、飞书任务、钉钉任务 |
| 授权方式 | OAuth 2.0 |
| 数据同步 | 每日自动同步 + 手动刷新 |
| 验收标准 | 1. 授权流程顺畅 2. 数据准确 3. 支持多账号 |

#### 功能 3：模板中心

| 项目 | 描述 |
|------|------|
| 功能描述 | 提供多岗位周报模板，用户可选择使用 |
| 模板类型 | 技术、产品、运营、销售、管理、自定义 |
| 模板结构 | 标题、正文结构、语气风格 |
| 验收标准 | 1. 模板丰富 2. 可自定义 3. 支持收藏 |

## 3. Prompt 设计方案

### 3.1 基础 Prompt

```
# Role
你是一名专业的职场写作助手，擅长撰写高质量周报。

# Task
根据用户提供的工作内容，生成一份专业、结构化的周报。

# Input
{work_content}

# Requirements
1. 结构：本周工作、遇到的问题、下周计划
2. 风格：专业、简洁、数据驱动
3. 字数：500-800 字
4. 突出工作价值和成果

# Output Format
## 本周工作总结
- 完成事项 1（数据/成果）
- 完成事项 2（数据/成果）

## 遇到的问题与解决方案
- 问题 1 → 解决方案
- 问题 2 → 解决方案

## 下周工作计划
- 计划 1（预期成果）
- 计划 2（预期成果）
```

### 3.2 岗位差异化 Prompt

| 岗位 | Prompt 调整重点 |
|------|---------------|
| 技术 | 突出技术难点、代码质量、性能优化 |
| 产品 | 突出需求进展、数据变化、用户反馈 |
| 运营 | 突出活动效果、转化数据、用户增长 |
| 销售 | 突出业绩完成、客户进展、回款情况 |
